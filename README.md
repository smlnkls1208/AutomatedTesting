# АВТОТЕСТЫ ДЛЯ ПЛАТФОРМЫ PROFTEAM

Проект автоматизированного тестирования веб-платформы https://dev.profteam.su с использованием Cypress.


## О ПРОЕКТЕ

Проект содержит End-to-End (E2E) автотесты для проверки ключевых сценариев взаимодействия:

- Студент - поиск вакансий, отклик, просмотр рабочих пространств
- Работодатель - создание вакансий, управление откликами, смена статусов

Тесты проверяют как успешные сценарии, так и негативные (ошибки валидации, блокировки, disabled-состояния).


## БЫСТРЫЙ СТАРТ

1. Клонирование репозитория:

git clone https://github.com/smlnkls1208/AutomatedTesting.git


2. Установка зависимостей:

npm install

3. Запуск тестов:

npx cypress open - Открыть Cypress Test Runner (выбор тестов в браузере)
npx cypress run - Запуск всех тестов в headless-режиме
npx cypress run --headed - Запуск всех тестов с открытым браузером
npx cypress run --spec "путь/к/тесту.cy.js" - Запуск конкретного теста

Примеры запуска:

npx cypress run --spec "cypress/e2e/vacancies/apply.cy.js"
npx cypress run --spec "cypress/e2e/vacancies/*.cy.js"

## ПОДРОБНОЕ ОПИСАНИЕ


# ПАПКА: cypress/


Главная директория, содержащая все файлы тестирования.

Назначение: Хранение всех компонентов автотестов (сами тесты, тестовые данные, вспомогательные скрипты).


# ПАПКА: cypress/e2e/


Директория с тестовыми сценариями.

Назначение: Содержит все тесты, разделённые по функциональным модулям. Каждый файл с расширением .cy.js представляет собой отдельный набор тестов.


# ПАПКА: cypress/e2e/vacancies/


Тесты модуля "Вакансии".

Файлы:

apply.cy.js - Отклик студента на вакансию

create.cy.js - Создание вакансии работодателем

responses.cy.js - Управление откликами (принять/отклонить)

search.cy.js - Поиск и фильтрация вакансий

status.cy.js - Смена статуса рабочего пространства

workspace-management.cy.js - Работа с рабочими пространствами


# ФАЙЛ: cypress/e2e/AutoTest.cy.js


Сборник/запуск всех тестов.

Назначение: Объединяет все тесты из папки vacancies для массового запуска или служит точкой входа для выполнения всего набора тестов.


# ПАПКА: cypress/fixtures/


Директория с тестовыми данными.

Назначение: Хранение статичных данных, используемых в тестах (JSON-файлы, текстовые файлы, изображения и т.д.).

Файлы:

user.json - Данные пользователей для тестирования (логины, пароли, роли)

vacansy.json - Шаблоны данных для создания/проверки вакансий (названия, описания, требования)


# ПАПКА: cypress/support/


Директория вспомогательных файлов.

Назначение: Содержит переиспользуемые команды и глобальные настройки для всех тестов.

Файлы:

commands.js - Кастомные команды Cypress (логин, авторизация по ролям, общие утилиты)

e2e.js - Глобальные хуки и конфигурация (выполняется перед каждым тестом, обработка ошибок, импорт команд)


ФАЙЛ: cypress.config.js


# Корневой конфигурационный файл Cypress.

Настройки:
- baseUrl - Базовый адрес тестируемого сайта
- specPattern - Путь к файлам с тестами
- supportFile - Путь к вспомогательному файлу
- defaultCommandTimeout - Таймаут выполнения команд
- viewportWidth/Height - Размер окна браузера
- screenshotOnRunFailure - Автоматическое создание скриншотов
- video - Запись видео прохождения тестов


# ФАЙЛ: package.json


Файл управления зависимостями Node.js.

Содержит:
- Список установленных пакетов (cypress)
- NPM скрипты для запуска тестов
- Метаданные проекта (имя, версия, автор)

## ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

Проблема: cypress: command not found
Решение: Установите зависимости командой npm install

Проблема: Тест падает с таймаутом
Решение: Увеличьте defaultCommandTimeout в cypress.config.js

Проблема: Не находится элемент на странице
Решение: Проверьте актуальные селекторы в браузере (F12 - Elements)

Проблема: Ошибка авторизации
Решение: Проверьте логин/пароль в commands.js

Проблема: Тест падает из-за uncaught:exception
Решение: В e2e.js добавлен обработчик Cypress.on('uncaught:exception', () => false)
